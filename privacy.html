สรุปสั้นๆ ก่อนอ่านเต็ม

Rundub เป็นแอปชุมชนนักวิ่งที่ออกแบบให้ เก็บข้อมูลเท่าที่จำเป็น เพื่อให้ฟีเจอร์ทำงาน — บันทึกการวิ่ง, แชร์กับเพื่อน, จัดลำดับสวน เราไม่ขายข้อมูลของคุณ และคุณสามารถ ลบบัญชี + ข้อมูลทั้งหมด ได้ทุกเมื่อในแอป

1. ข้อมูลที่เราเก็บ

1.1 ข้อมูลบัญชี

  • เบอร์โทรศัพท์ — สำหรับยืนยันตัวตน (Phone OTP) เก็บแบบ hashed ผ่าน Supabase Auth
  • Username — เลือกตอน onboarding, ใช้แสดงในระบบ
  • ชื่อที่แสดง + bio — ตั้งใน Settings (ไม่บังคับ)
  • รูปโปรไฟล์ (avatar) — อัพโหลดเอง (ถ้าตั้ง)

1.2 ข้อมูลการวิ่ง

  • ระยะทาง, ระยะเวลา, เพซ, อัตราการเต้นหัวใจ (ถ้ามี), แคลอรี่
  • เส้นทาง GPS (จุด lat/lon ทุก ~2-5 เมตร) เมื่อใช้ "เริ่มวิ่ง" ในแอป
  • ความสูง (elevation gain) เมื่อมีข้อมูลจาก GPS
  • สวน (park) ที่ระบบตรวจจับอัตโนมัติ หรือคุณเลือกตอน save
  • เวลาเริ่ม/จบของแต่ละการวิ่ง

1.3 ข้อมูลตำแหน่ง

  • Foreground (ขณะใช้แอป) — สำหรับตรวจจับสวนตอน save run
  • Background (ตลอดเวลา, optional) — สำหรับบันทึก GPS เมื่อล็อกหน้าจอ ตอนวิ่ง คุณเลือกอนุญาตหรือไม่ก็ได้
  • Live broadcast — ถ้าคุณเปิด เราจะแชร์ตำแหน่งปัจจุบันให้คนที่คุณอนุญาต (ทุกคน / followers) ขณะวิ่ง — หยุดทันทีเมื่อคุณกดหยุด

1.4 ข้อมูลการใช้งานสังคม

  • การ follow / ถูก follow
  • การกด "เริ่ด" (like), comment, mention (@username) บนการวิ่งของผู้อื่น
  • การเข้าร่วมกลุ่ม, สร้าง/ร่วม challenge
  • การ block ผู้อื่นจาก live tracking

1.5 ข้อมูลอุปกรณ์

  • Push notification token (สำหรับส่งแจ้งเตือนผ่าน FCM)
  • Platform (Android / iOS)
  • App version

1.6 ข้อมูลจาก third-party (เมื่อคุณเชื่อมต่อ)

  • Strava — กิจกรรมวิ่งย้อนหลังที่คุณเลือก import (รวมเส้นทาง + HR + pace)
  • Health Connect (Android) — กิจกรรมวิ่งจาก Samsung Health / Garmin / Fitbit / อื่นๆ ที่คุณเชื่อมไปยัง Health Connect

2. วิธีที่เราใช้ข้อมูล

  • แสดงข้อมูลของคุณในแอป (Profile, Feed, Calendar, Run history)
  • คำนวณ leaderboard ของแต่ละสวน, badges, achievements
  • ส่ง push notification เมื่อมีคนกดเริ่ดให้, comment, mention, follow คุณ
  • ตรวจจับสวนที่คุณวิ่ง (ผ่าน PostGIS spatial query บน GPS)
  • กรอง runs ตาม privacy setting (everyone / followers / private)
  • ปรับปรุงและ debug แอป (เห็น crash reports ผ่าน EAS Insights ไม่มี personal data)

เราไม่ใช้ข้อมูลของคุณเพื่อ:

  • ขายให้บุคคลที่สาม
  • โฆษณา targeted ads
  • ส่ง marketing emails / SMS โดยไม่ได้ขออนุญาต
  • วิเคราะห์เพื่อสร้าง profile ของคุณนอกแอป

3. ใครเห็นข้อมูลของคุณบ้าง

3.1 ผู้ใช้แอปคนอื่น (ตาม privacy setting)

  • Everyone — ทุกคนใน Rundub เห็น (รวมถึงในสวน leaderboard)
  • Followers — เฉพาะคนที่ follow คุณ
  • Private — เฉพาะคุณ (ไม่นับใน leaderboard)

3.2 Service providers (เก็บ/ประมวลผลให้เรา)

  • Supabase (https://supabase.com) — เก็บฐานข้อมูล (PostgreSQL) + auth + storage (avatars) · data center: AWS
  • Vercel (https://vercel.com) — โฮสต์หน้า redirect และ legal pages
  • Expo (https://expo.dev) — จัดส่ง push notifications + OTA updates
  • Firebase Cloud Messaging (Google) — ส่ง push บน Android
  • Twilio (เมื่อ live) — ส่ง SMS OTP (Phase 2)

3.3 Third-party integrations (เฉพาะที่คุณเชื่อมต่อเอง)

  • Strava — เราดึงข้อมูลกิจกรรมตาม scope ที่คุณอนุญาต (activity:read_all). เรา ไม่ เขียนข้อมูลกลับไปยัง Strava
  • Health Connect — เราอ่าน Exercise / Distance / HeartRate / Calories ตามที่คุณอนุญาต. ข้อมูลอยู่ในเครื่องของคุณก่อนเข้าระบบเรา

4. ความปลอดภัยของข้อมูล

  • การส่งข้อมูลทั้งหมดผ่าน HTTPS / TLS
  • เก็บ password (จาก Phone OTP) ผ่าน Supabase Auth (bcrypt + best practices)
  • RLS (Row-Level Security) บน Supabase — user ของคุณเข้าถึงได้แค่ data ของตัวเอง
  • Strava OAuth tokens เก็บแบบ encrypted at rest บน Supabase

5. การเก็บรักษาข้อมูล

  • ข้อมูลถูกเก็บตราบใดที่บัญชีของคุณยังใช้งาน
  • เมื่อคุณ ลบบัญชี (Settings → ออกจากระบบ + ลบข้อมูล สำหรับ Guest user, หรือกดลบบัญชีถาวรสำหรับ phone-linked) — ข้อมูล run / follows / likes / comments / avatar / Strava tokens จะถูกลบใน 30 วัน
  • ข้อมูลบางส่วน (เช่น log) อาจถูกเก็บนานกว่าตามข้อบังคับทางกฎหมาย

6. สิทธิ์ของคุณ (ตาม PDPA)

ภายใต้พระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 คุณมีสิทธิ์:

  • เข้าถึงข้อมูล — ขอสำเนาข้อมูลของคุณที่เราเก็บ
  • แก้ไข — แก้ profile / runs ผ่านในแอป หรือขอแก้ผ่านการติดต่อ
  • ลบ — ลบบัญชีและข้อมูลทั้งหมด
  • โอนย้าย — Export ข้อมูล (ขอผ่านอีเมล)

Quick summary

Rundub is a running community app designed to collect only what's needed for features to work — recording runs, sharing with friends, ranking by park. We do not sell your data, and you can delete your account + all data at any time from the app.

1. Information we collect

1.1 Account data

  • Phone number — for OTP verification (hashed via Supabase Auth)
  • Username — chosen during onboarding
  • Display name + bio — optional (set in Settings)
  • Avatar image — optional (you upload)

1.2 Run data

  • Distance, duration, pace, heart rate (if available), calories
  • GPS route (lat/lon points every ~2-5 meters) when using in-app recording
  • Elevation gain when available
  • Park (auto-detected via GPS or chosen at save)
  • Start/end timestamps

1.3 Location data

  • Foreground (while using app) — for park auto-detection at save
  • Background (always, optional) — for GPS recording when phone is locked; you choose to grant or not
  • Live broadcast — when enabled, we share your current location with people you allow (everyone/followers) during a run, stopping immediately when you stop

1.4 Social activity

  • Follows / followers
  • Likes (เริ่ด), comments, mentions (@username) on others' runs
  • Group memberships, challenge creation/participation
  • Blocked users (for live tracking)

1.5 Device data

  • Push notification token (for FCM delivery)
  • Platform (Android / iOS)
  • App version

1.6 Third-party data (when you connect)

  • Strava — past activities you choose to import (including route, HR, pace)
  • Health Connect (Android) — runs from Samsung Health, Garmin, Fitbit, etc. routed through Health Connect

2. How we use it

  • Display your information in the app (Profile, Feed, Calendar, Run history)
  • Compute park leaderboards, badges, achievements
  • Send push notifications when someone likes, comments, mentions, or follows you
  • Auto-detect park where you ran (PostGIS spatial query on GPS)
  • Filter runs by privacy setting (everyone / followers / private)
  • Improve and debug the app (crash reports via EAS Insights — no personal data)

We do not use your data for:

  • Selling to third parties
  • Targeted advertising
  • Marketing emails / SMS without consent
  • Profile building outside the app

3. Who sees your data

3.1 Other app users (per your privacy setting)

  • Everyone — visible to all Rundub users (including park leaderboards)
  • Followers — only to people who follow you
  • Private — only to you (excluded from leaderboards)

3.2 Service providers (store/process for us)

  • Supabase — database (PostgreSQL), auth, storage · AWS data center
  • Vercel — hosts redirect + legal pages
  • Expo — push notification dispatch + OTA updates
  • Firebase Cloud Messaging (Google) — Android push delivery
  • Twilio (when live) — SMS OTP delivery (Phase 2)

3.3 Third-party integrations (when you connect)

  • Strava — we fetch activities per your granted scope (activity:read_all). We do NOT write back to Strava.
  • Health Connect — we read Exercise, Distance, HeartRate, Calories as authorized. Data lives on your device first.

4. Security

  • All transit over HTTPS / TLS
  • Authentication via Supabase Auth (bcrypt + industry best practices)
  • Row-Level Security (RLS) — your user can only access your own data
  • Strava OAuth tokens stored encrypted at rest

5. Data retention

  • Data is kept as long as your account is active
  • When you delete your account, your runs, follows, likes, comments, avatar, and Strava tokens are deleted within 30 days
  • Some data (e.g., logs) may be retained longer per legal requirements

6. Your rights (under PDPA Thailand & GDPR-equivalent)

You have the right to:

  • Access — request a copy of your data
  • Rectify — edit profile/runs in app or by contact
  • Erase — delete your account and all data
  • Port — export your data (email request)
  • Object to processing
  • Withdraw consent — e.g., disable push notifications in Settings

Contact noppawit.p@apprundub.com for rights requests.
